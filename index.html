Apresentação

Nesta aula prática serão apresentados os seguintes conceitos relacionados à camada de enlace de redes de computadores:

  • Elementos para interligação de redes
  • Domínio de colisão
  • Domínio broadcast
  • Endereço broadcast
  • Rede local virtual (VLAN)
  • Protocolo ARP

Elementos para interligação de redes

Repetidor

  • Interliga níveis físicos diferentes com mesmo MAC (método de controle de acesso).
  • Os dispositivos ligados através de um repetidor se encontram em um mesmo domínio de colisão e de broadcast.

Roteador

  • Permite interligar redes diferentes
  • Divide um dominio de broadcast em dois ou mais domínios de broadcast menores.

Switch

  • Capaz de selecionar em que porta os frames devem ser enviados.
  • Divide um dominio de colisão em dois ou mais domínios de colisão menores.
  • Cada porta do switch corresponde a um domínio de colisão diferente.

Domínio de colisão

O domínio de colisão é um segmento lógico da rede onde os pacotes transmitidos por elementos pertencentes a ele podem colidir uns com os outros.

Domínio broadcast

O domínio broadcast consiste em um conjunto de dispositivos que recebem qualquer pacote broadcast originário de qualquer dispositivo dentro do segmento de rede.

Endereço broadcast

Um endereço de destino MAC broadcast é definido por FF:FF:FF:FF:FF:FF, e significa que o pacote deve ser enviado de um host origem para todos os outros hosts pertencentes à mesma rede.

Rede Local Virtual

  • Uma VLAN (Virtual Local Area Network - Rede Local Virtual) é uma rede local que reúne um conjunto de máquinas de maneira lógica
  • Define um domínio broadcast para pacotes do nível de rede
  • Definição de LANs separadas através da especificação de um subconjunto de portas em um switch ou portas em vários switches
  • O switch funciona como se fossem switches separados
  • Para interligar diferentes VLANs é necessário um elemento a mais, o roteador.
  • O roteador também pode estar implementado no próprio switch (não ocupa uma porta em cada VLAN que interliga)

Protocolo ARP

O protocolo ARP (Address Resolution Protocol) é responsável pela conversão de um endereço IP em um endereço MAC.

Assim, para fazer a correspondência entre os endereços físicos e os endereços lógicos, o protocolo ARP interroga as máquinas da rede para conhecer o seu endereço físico e depois cria uma tabela de correspondência entre os endereços lógicos e os endereços físicos numa memória secreta.
